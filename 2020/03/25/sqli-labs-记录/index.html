。。看了下，login.php 的测试点好玩些

使用 whitelist 对 $id 参数进行拦截，当输入参数非数字，则跳转到 hacked.php

当匹配到非数字参数进行 Location 跳转时，并没有及时终止原有程序，恶意参数仍会被带入程序接下来的 SQL 查询语句中

其余无任何过滤，使用 BurpSuite 抓包测试

MYSQL 注入天书提到此处还涉及参数污染，服务器端分为两个部分：

  • 基于 Tomcat 引擎的 JSP 服务器，负责数据过滤和处理（类似 WAF）
  • 基于 Apache 引擎的 PHP 服务器，负责提供 Web 服务

由于两层服务器存在解析差异，因此可以进行 HPP（HTTP Parameter Pollution） 攻击，例如提交参数 ?id=1&id=2

客户端请求首先通过 Tomcat，Tomcat 对第一个参数 id=1 进行过滤和处理，接下来 Tomcat 去请求 Apache 服务器，Apache 引擎则会解析最后一个参数 id=2，Web 服务由 Apache 引擎提供，因此返回结果为 id=2 的结果
